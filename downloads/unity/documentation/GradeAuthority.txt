TALVAREN STUDIOS
GRADE AUTHORITY
INSTALLATION, MANUAL, LICENSE, AND SUPPORT

OVERVIEW
Grade Authority is part of the TerrainVale Suite for the Unity Editor.

Grade Authority is a terrain behavior authoring tool designed to establish meaningful environmental constraints through slope relationships rather than artificial restrictions.

PROBLEM SOLVED
Modern games often rely on invisible barriers and inconsistent terrain sculpting to guide players. Grade Authority establishes believable environmental constraints through meaningful slope relationships.

REQUIREMENTS
- Unity Editor
- Unity Terrain System
- Existing Unity Terrain

INSTALLATION
1. Back up the Unity project before importing any editor extension.
2. Open the target project in a supported Unity Editor installation.
3. Import the supplied .unitypackage through Assets > Import Package > Custom Package.
4. Review the package contents, then complete the import.
5. Allow Unity to compile before opening the tool.
6. Use the tool's own creation commands to establish any required project folders or data structures.

PRIMARY FEATURES
- Target grade authoring using meaningful slope values
- Adjustable grade variance for natural results
- Convergence-based terrain modification
- Average grade measurement and feedback
- Controlled approach toward desired outcomes
- Support for traversal design and environmental storytelling
- Reduces reliance on invisible barriers
- Encourages natural environmental readability

WORKFLOW ROLE
Grade Authority teaches the environment what traversal possibilities exist. Following Terrain, Territory, Path, and Ribbon Authority, it establishes believable slopes and environmental constraints before Paint Authority completes the visual language.

USE AND PROJECT SAFETY
- Test the tool in a duplicate project or source-controlled branch before applying changes to production content.
- Save the project before terrain generation, conversion, painting, or destructive editor operations.
- Confirm terrain resolution, dimensions, layers, neighboring terrain relationships, and project settings before processing.
- Large terrains and high resolutions may require additional processing time and memory.
- Results depend on project configuration, Unity version, terrain data, and author-selected settings.
