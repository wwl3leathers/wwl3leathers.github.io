TALVAREN STUDIOS
TERRITORY AUTHORITY
INSTALLATION, MANUAL, LICENSE, AND SUPPORT

OVERVIEW
Territory Authority is part of the TerrainVale Suite for the Unity Editor.

Territory Authority transforms large Unity terrains into standardized, interconnected territories designed for scalable world construction.

PROBLEM SOLVED
Large terrains become increasingly difficult to organize and maintain. Territory Authority converts existing Unity terrains into standardized territory grids while preserving visual continuity.

REQUIREMENTS
- Unity Editor
- Unity Terrain System
- Existing Unity Terrain

INSTALLATION
1. Back up the Unity project before importing any editor extension.
2. Open the target project in a supported Unity Editor installation.
3. Import the supplied .unitypackage through Assets > Import Package > Custom Package.
4. Review the package contents, then complete the import.
5. Allow Unity to compile before opening the tool.
6. Use the tool's own creation commands to establish any required project folders or data structures.

PRIMARY FEATURES
- Converts large terrains into standardized territory grids
- Multiple territory subdivision configurations
- Preserves terrain continuity through height reconstruction
- Automatically establishes neighboring terrain relationships
- Maintains visual cohesion across territory boundaries
- Preserves the original terrain by disabling rather than destroying it

WORKFLOW ROLE
Territory Authority is the organizational foundation of the VALE workflow. It divides large terrains into standardized, connected territories and prepares the world for Path, Ribbon, Grade, and Paint Authority.

PACKAGE CONTENTS
- One self-contained Unity .unitypackage containing the tool code.
- This single plain-text documentation file.
- No sample scenes, sample terrains, screenshots, bundled videos, changelogs, version folders, or unrelated assets.

USE AND PROJECT SAFETY
- Test the tool in a duplicate project or source-controlled branch before applying changes to production content.
- Save the project before terrain generation, conversion, painting, or destructive editor operations.
- Confirm terrain resolution, dimensions, layers, neighboring terrain relationships, and project settings before processing.
- Large terrains and high resolutions may require additional processing time and memory.
- Results depend on project configuration, Unity version, terrain data, and author-selected settings.
